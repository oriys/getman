# Getman 功能 Roadmap（未来 12 个月）

时间范围：2026 年 3 月 - 2027 年 2 月  
定位：`Postman` 轻量级替代，桌面优先（`iced`），先把 HTTP 请求开发体验做到快、稳、可复用。

## 产品目标

1. `MVP` 到 `1.0`：覆盖日常 API 调试的核心闭环（请求构建 -> 发送 -> 调试 -> 复用 -> 自动化）。
2. 性能目标：普通 JSON 响应（< 2MB）渲染延迟 < 150ms。
3. 稳定性目标：核心请求流程崩溃率接近 0，错误可定位。
4. 易用性目标：新用户 5 分钟内完成第一次可复用请求。

## 里程碑总览

| 阶段 | 时间 | 版本目标 | 关键结果 |
|---|---|---|---|
| Phase 1 | 2026.03 - 2026.05 | v0.1 - v0.3 | 完成 HTTP 基础能力与可用 UI |
| Phase 2 | 2026.06 - 2026.08 | v0.4 - v0.6 | 建立 Collection/Environment 体系 |
| Phase 3 | 2026.09 - 2026.11 | v0.7 - v0.9 | 引入自动化测试与批量运行 |
| Phase 4 | 2026.12 - 2027.02 | v1.0 | 稳定化、生态化与发布 1.0 |

## Phase 1（2026.03 - 2026.05）：HTTP 核心可用

1. 请求构建能力
- 方法/URL/Headers/Body 全量编辑
- 查询参数编辑器（Key-Value）
- Body 模式：raw / JSON / form-data / x-www-form-urlencoded

2. 响应调试能力
- 状态码、耗时、响应大小
- 响应头/响应体分区展示
- JSON 格式化、折叠、复制

3. 交互体验
- 多标签请求页
- 请求历史（最近 100 条）
- 错误提示标准化（网络、超时、TLS、DNS）

4. 工程保障
- 崩溃恢复（启动时恢复上次未保存输入）
- 基础日志与诊断信息导出

## Phase 2（2026.06 - 2026.08）：可复用与协作基础

1. Collection 管理
- 目录树、请求分组、重命名/复制/拖拽
- 本地持久化与快速搜索

2. Environment/变量系统
- 全局变量、环境变量、集合变量
- `{{variable}}` 插值与预览
- 多环境切换（dev/staging/prod）

3. 认证能力
- Bearer Token
- Basic Auth
- API Key（Header/Query）
- OAuth2（先支持常见授权码流程）

4. 兼容能力
- Postman Collection v2.1 导入（首版）
- Getman Project 导出/导入

## Phase 3（2026.09 - 2026.11）：自动化测试

1. 断言与测试
- 响应状态/头/body JSONPath 断言
- 测试结果可视化（通过/失败/耗时）
- 失败样本快速定位

2. 批量运行
- Collection Runner（串行/并行）
- 变量数据集驱动运行（CSV/JSON）
- 汇总报告（成功率、失败原因分布）

3. CLI 初版
- `getman run collection.json --env prod`
- CI 场景的退出码与报告文件输出

4. 安全与权限
- 敏感字段脱敏展示
- 本地密钥加密存储（token、secret）

## Phase 4（2026.12 - 2027.02）：1.0 稳定发布

1. 1.0 稳定化
- 核心模块重构与性能优化
- 大响应体渲染优化（流式/分页）
- 大规模 Collection 打开性能优化

2. 扩展能力
- 插件机制（请求前后钩子）
- 自定义脚本扩展点（受限沙箱）

3. 协作与发布
- 项目级文件结构定版
- 团队共享模板（请求模板、断言模板）
- 发布 `v1.0.0` 与升级迁移文档

## 版本节奏（建议）

1. 每月一个小版本（`v0.x`）。
2. 每个季度末一个里程碑版本（`v0.3/v0.6/v0.9/v1.0`）。
3. 所有新功能均要求：
- 至少 1 个端到端可验证用例
- 关键路径回归测试通过
- 有对应用户文档或示例

## 风险与应对

1. `iced` 生态变化带来的 API 变动
- 应对：固定版本，季度统一升级。

2. HTTP 复杂场景兼容性（代理、证书、超时）
- 应对：先覆盖 80% 常见场景，逐步补齐高级配置。

3. 功能膨胀导致“轻量”丢失
- 应对：默认体验极简，高级功能放在可选面板。

## 成功指标（2027.02 检查）

1. 日常 API 调试核心功能覆盖率 >= 90%。
2. 冷启动时间 < 2 秒（常见机器）。
3. 单次请求配置完成时间较当前版本缩短 40% 以上。
4. `v1.0` 发布后 4 周内无 P0/P1 严重缺陷。
